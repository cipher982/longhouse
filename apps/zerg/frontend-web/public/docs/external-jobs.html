3

Register in manifest.py

# manifest.py
from zerg.jobs import job_registry, JobConfig
from jobs.daily_report import run as daily_report

job_registry.register(JobConfig(
    id="daily-report",
    cron="0 8 * * *",           # 8 AM UTC daily
    func=daily_report,
    timeout_seconds=300,        # 5 minute timeout
    max_attempts=3,             # Retry up to 3 times
    tags=["reports"],
    project="my-project",
    description="Daily performance report",
))

Key Features

Durable Queue

Jobs are stored in a database queue. If Swarmlet restarts mid-execution, incomplete jobs are automatically retried. No duplicate runs thanks to deduplication by job_id + scheduled_time.

Automatic Retries

Failed jobs retry with exponential backoff: immediate, +1min, +2min, +4min... capped at 1 hour between retries.

Version Tracking

Every job run records the git SHA that was executing, so you always know exactly which code version ran.
